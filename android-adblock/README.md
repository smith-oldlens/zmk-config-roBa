# roBa AdBlock — Android用 広告ブロッカー

Android のブラウザ（Chrome など）やアプリに表示される広告を、**端末全体で自動的にブロックする**アプリです。root 不要・外部サーバー不要で動作します。

## 仕組み

Chrome for Android は拡張機能に対応していないため、DNS66 や AdGuard と同じ「ローカル VPN 方式」を採用しています。

1. Android の `VpnService` API でローカル VPN を張ります。
2. ただしルーティングするのは **仮想 DNS サーバー宛の通信 (10.111.222.53) だけ**です。Web ページ・動画・ゲームなどの実データは VPN を一切通らず、通常のネットワークをそのまま流れます。→ **速度低下や他機能への影響がほぼありません。**
3. アプリが DNS 問い合わせを検査し、ドメインが `blocklist.txt` に載っていれば NXDOMAIN(存在しないドメイン)を返して広告の読み込み自体を止めます。
4. それ以外の問い合わせは通常のリゾルバ (1.1.1.1) にそのまま転送します。
5. 通信内容が外部サーバーに送られることはありません。すべて端末内で処理されます。

```
アプリ/ブラウザ ──DNS問い合わせ──▶ ローカルVPN(本アプリ)
                                   ├─ 広告ドメイン → NXDOMAIN を返す(ブロック)
                                   └─ それ以外   → 1.1.1.1 へ転送
その他すべての通信 ─────────────▶ 通常のネットワーク(VPNを通らない)
```

## インストール方法

1. GitHub リポジトリの **Actions** タブ → 「**Build AdBlock APK**」ワークフローの最新の実行を開く
2. 下部の **Artifacts** から `roba-adblock-debug-apk` をダウンロード
3. zip を展開して `app-debug.apk` をスマホに転送(または スマホのブラウザで直接ダウンロード)し、タップしてインストール
   - 「提供元不明のアプリ」の許可を求められたら許可してください
4. アプリを開いて「**ブロックを開始**」をタップ → VPN 接続の確認ダイアログで「OK」

以後、通知バーに鍵アイコンが表示されている間は広告ブロックが有効です。ワンタップで停止できます。

## ブロックリストのカスタマイズ

`app/src/main/assets/blocklist.txt` にドメインを 1 行 1 つで追加できます。サブドメインは自動的にブロック対象になります(`doubleclick.net` を書けば `ad.doubleclick.net` もブロック)。編集後に push すると GitHub Actions が新しい APK をビルドします。

リストは「他の機能を壊さない」ことを優先した保守的な内容にしてあります(ログイン・地図・動画再生などに使われるドメインは含めていません)。

## 制限事項

- **YouTube アプリ内の動画広告はブロックできません**(広告が動画と同じドメインから配信されるため。これはDNS方式のブロッカー共通の制限です)
- IPv4 の DNS のみ処理します。まれに IPv6 のみのネットワークでは効果が下がることがあります
- 他の VPN アプリとは同時に使えません(Android の仕様で VPN は 1 つだけ)
- Chrome の「保護されたDNS(Secure DNS)」が有効だと DNS がアプリを迂回することがあります。効果が薄い場合は Chrome の設定 → プライバシーとセキュリティ → セキュア DNS をオフにしてください

## アプリを作らない簡単な代替手段

コード不要で今すぐ試せる方法もあります:

- **プライベート DNS(推奨・設定1分)**: 設定 → ネットワークとインターネット → プライベート DNS → 「`dns.adguard-dns.com`」を入力。端末全体で広告ドメインがブロックされます(このアプリと同等の効果)
- **Brave ブラウザ / Firefox + uBlock Origin**: ブラウザを替えるだけで、ページ内広告をより強力にブロックできます
