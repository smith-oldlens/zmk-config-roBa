# Scene Composer — 風景写真に人物を小さく合成するWebアプリ

森や桜並木などの風景写真の「指定した場所」に、白ワンピースの女の子などの人物を
小さく入れ込んだ作品写真を作れる、スマホ対応のWebアプリです。

- 人物は **Gemini の画像生成API**（あなたのAPIキー）で作ります
- 切り抜き・配置・色なじませ・影付け・書き出しは **すべてブラウザ内** で処理します
  （写真がサーバーにアップロードされることはありません）
- ビルド不要の静的サイト（HTML/CSS/JSのみ）

## 使い方

1. **APIキーを設定**（初回のみ）
   - [Google AI Studio](https://aistudio.google.com/apikey) で無料のAPIキーを取得
   - アプリ右上の ⚙ からキーを貼り付けて保存（端末内にのみ保存されます）
2. **風景写真を選ぶ**（ファイル選択 or ドラッグ＆ドロップ）
3. **✨人物を生成** — プリセット（「白ワンピの女性（後ろ姿）」など）を選ぶか、
   自由に説明を入力して生成。緑背景で生成された人物が自動で切り抜かれます。
   - うまく抜けないときは「切り抜きの調整」で抜き具合を変えるか、
     スポイトで背景色を指定し直せます
4. **写真をタップして配置** — 置きたい場所（人物の足元）をタップ
   - ドラッグで移動、ピンチまたは「大きさ」スライダーで拡大縮小
   - 「色なじみ」で周囲の色調・明るさに自動マッチ、影の濃さ・ぼかし・向きも調整可能
5. **⬇保存** — 元の写真と同じ解像度でJPEG/PNGに書き出し（共有シート対応端末では共有も可）

生成した人物は「👤ライブラリ」に自動保存され、別の写真でも再利用できます。

## ローカルで動かす

ES Modules を使っているため、`file://` では動きません。簡易サーバーで配信してください：

```bash
cd photo-composer
python3 -m http.server 8000
# → http://localhost:8000 を開く
```

## GitHub Pages で公開する

このリポジトリには `.github/workflows/pages.yml` が含まれており、
`main` ブランチの `photo-composer/` が変更されると自動でデプロイされます。

初回のみリポジトリの設定が必要です：

1. GitHub のリポジトリページ → **Settings → Pages**
2. **Source** を **GitHub Actions** に変更

公開URL: `https://smith-oldlens.github.io/zmk-config-roBa/`

## APIキーの取り扱いについて

- キーはブラウザの `localStorage` にのみ保存され、Google の API への送信以外には使われません
- **キーをリポジトリにコミットしないでください**
- 公開ページで使う場合は、Google AI Studio 側でキーに
  **HTTPリファラ制限**（自分のPagesのURLのみ許可）を掛けておくと安全です
- Gemini API の無料枠を超えると課金が発生する場合があります（画像生成は1枚ごとに消費）

## 技術メモ

| ファイル | 役割 |
|---|---|
| `index.html` | UIレイアウト |
| `style.css` | スマホファーストのスタイル |
| `app.js` | 状態管理・タップ配置・ドラッグ/ピンチ・Undo・IndexedDBライブラリ・書き出し |
| `gemini.js` | Gemini画像生成API（`gemini-2.5-flash-image`）呼び出し |
| `compositor.js` | クロマキー透過・色調マッチ（簡易color transfer）・影の描画・合成 |

- 人物は「均一な緑背景（#00FF00）」指定で生成し、ブラウザ内でクロマキー処理して透過化
- 色なじませは配置先周辺の背景ピクセル統計（平均・分散）への簡易 Reinhard color transfer
- 影は人物シルエットを潰して足元に投影＋接地影の楕円
- 編集中のプレビューは長辺2048pxに縮小し、書き出し時のみ元解像度で再合成（メモリ対策）
