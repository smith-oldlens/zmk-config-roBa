# Scene Composer — 風景写真に人物を小さく合成するWebアプリ

森や桜並木などの風景写真の「指定した場所」に、白ワンピースの女の子などの人物を
小さく入れ込んだ作品写真を作れる、スマホ対応のWebアプリです。

- 人物は **画像生成AI** で作ります。**デフォルトは Pollinations（無料・APIキー不要）** なので、
  何も設定せずにそのまま生成できます。より高品質にしたい場合は Google Gemini（要APIキー・有料枠）も選べます
- 切り抜き・配置・色なじませ・影付け・書き出しは **すべてブラウザ内** で処理します
  （写真がサーバーにアップロードされることはありません）
- ビルド不要の静的サイト（HTML/CSS/JSのみ）

## 対応している画像生成サービス

| サービス | モデル | キー | 費用 | 状態 |
|---|---|---|---|---|
| **Pollinations** | Flux | **不要** | **無料** | ✅ 有効（既定） |
| Google Gemini | `gemini-2.5-flash-image` | [AI Studio](https://aistudio.google.com/apikey) で取得 | 有料枠（お支払い設定）が必要※ | ✅ 有効 |
| OpenAI | `gpt-image-1` | [OpenAI](https://platform.openai.com/api-keys) で取得 | 従量課金＋組織の本人確認が必要 | 実装済み・**一旦無効** |

- Pollinations と Gemini は緑背景で生成し、ブラウザ内でクロマキー透過します
- ※ 2025年12月以降、Gemini の無料枠では画像生成の枠が0のため、無料のままだと `429` エラーになります
- OpenAI を有効化するには `photo-composer/providers.js` の以下2箇所のコメントを外すだけです：

```js
import { openai } from './openai.js';                      // ← コメントを外す
export const PROVIDERS = [pollinations, gemini, openai];   // ← openai を含める
```

新しいサービスを足したいときも、`pollinations.js` / `gemini.js` と同じ形の
プロバイダーオブジェクトを作って `providers.js` の `PROVIDERS` に追加するだけです。
（`noApiKey: true` を付ければキー不要サービスとして扱われます）

## 人物の用意のしかた（課金なしで使えます）

1. **外部の無料AIで生成 → 取り込む（おすすめ・完全無料）**
   - 「✨人物を生成」→「🤖 外部の無料AIで生成して取り込む手順」を開き、**「📋 生成用プロンプトをコピー」**
   - [Google AI Studio](https://aistudio.google.com)（無料）や Copilot に貼り付けて生成
   - できた画像をコピー → アプリに戻って **「📋 貼り付けて取り込む」**（PCなら Ctrl/⌘+V だけでもOK）
   - スマホなら画像を保存して**共有シートから「Scene Composer」へ共有**するだけで取り込めます（PWAインストール後）
2. **手持ちの画像を取り込む（完全無料・APIキー不要）**
   - 「🖼 画像を取り込む」から選択。透過PNGはそのまま、背景つき写真はスポイトで背景を抜けます
3. **Pollinations で自動生成（無料・APIキー不要）**
   - 既定のサービス。※ 提供元の状況により一時的に `403` 等で使えないことがあります
4. **Google Gemini で自動生成（高品質・要APIキー＆有料枠）**
   - ⚙設定で選択。画像生成に有料枠（お支払い設定）の有効化が必要（無料枠だと `429`）

## スマホでアプリのように使う（PWA）

GitHub Pages で公開すると、スマホから以下ができます：

- **ホーム画面に追加**：Android Chrome は「ホーム画面に追加」、iPhone Safari は共有 →「ホーム画面に追加」。全画面のアプリとして起動します
- **共有シートで受け取り**（Android）：写真アプリやAI Studioで生成した画像を「共有 → Scene Composer」に送ると、「人物として取り込む／背景写真にする」を選んで直接読み込めます
- 一度開けばオフラインでも起動します（生成以外の編集・書き出しはすべて端末内処理）

## 使い方

1. **人物を用意** — 上記のいずれかの方法で（迷ったら 1 の「外部AI生成→貼り付け」）
2. **風景写真を選ぶ**（ファイル選択 or ドラッグ＆ドロップ）
3. **✨人物を生成** — プリセット（「白ワンピの女性（後ろ姿）」など）を選ぶか、
   自由に説明を入力して生成。緑背景で生成された人物が自動で切り抜かれます。
   - うまく抜けないときは「切り抜きの調整」で抜き具合を変えるか、
     スポイトで背景色を指定し直せます
4. **写真をタップして配置** — 置きたい場所（人物の足元）をタップ
   - ドラッグで移動、ピンチまたは「大きさ」スライダーで拡大縮小
   - 「色なじみ」で周囲の色調・明るさに自動マッチ、影の濃さ・ぼかし・向きも調整可能
5. **⬇保存** — 元の写真と同じ解像度でJPEG/PNGに書き出し（共有シート対応端末では共有も可）

生成した人物は「👤ライブラリ」に自動保存され、別の写真でも再利用できます。

## ローカルで動かす

ES Modules を使っているため、`file://` では動きません。簡易サーバーで配信してください：

```bash
cd photo-composer
python3 -m http.server 8000
# → http://localhost:8000 を開く
```

## GitHub Pages で公開する

このリポジトリには `.github/workflows/pages.yml` が含まれており、
`main` ブランチの `photo-composer/` が変更されると自動でデプロイされます。

初回のみリポジトリの設定が必要です：

2. **Source** を **GitHub Actions** に変更

## APIキーの取り扱いについて

- キーはブラウザの `localStorage` にのみ保存され、選んだサービスの API への送信以外には使われません
- **キーをリポジトリにコミットしないでください**
- 公開ページで使う場合の安全対策：
  - **Gemini**：Google AI Studio / Cloud Console 側でキーに **HTTPリファラ制限**（自分のPagesのURLのみ許可）を掛ける
  - **OpenAI**：OpenAI のキーはブラウザに直接置くとリファラ制限が効かないため、**使用上限（Usage limits）を低めに設定**しておくと安心です。心配な場合は個人の端末でのローカル利用に留めてください
- 無料枠を超えると課金が発生する場合があります（画像生成は1枚ごとに消費。特に OpenAI は従量課金）

## 技術メモ

| ファイル | 役割 |
|---|---|
| `index.html` | UIレイアウト |
| `style.css` | スマホファーストのスタイル |
| `app.js` | 状態管理・タップ配置・ドラッグ/ピンチ・Undo・IndexedDBライブラリ・書き出し |
| `providers.js` | 画像生成プロバイダーの登録・切り替え・APIキー管理 |
| `pollinations.js` / `gemini.js` / `openai.js` | 各サービスの生成API呼び出し（プロバイダー実装） |
| `compositor.js` | クロマキー透過・色調マッチ（簡易color transfer）・影の描画・合成 |
| `manifest.webmanifest` / `sw.js` / `icons/` | PWA（ホーム画面追加・オフライン・共有シート受け取り） |

- Gemini は「均一な緑背景（#00FF00）」指定で生成し、ブラウザ内でクロマキー処理して透過化。
  OpenAI は `background: transparent` で透過PNGを直接取得
- 色なじませは配置先周辺の背景ピクセル統計（平均・分散）への簡易 Reinhard color transfer
- 影は人物シルエットを潰して足元に投影＋接地影の楕円。遠景ボケはプレビュー/書き出しで同一に見えるよう調整
- 編集中のプレビューは長辺2048pxに縮小。書き出しは元解像度だが、端末のCanvas制限を
  超える巨大写真は安全な範囲へ自動縮小（真っ黒化・失敗を防止）
